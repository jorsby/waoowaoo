## 一、现有数据结构分析

### 1.1 角色数据 — `NovelPromotionCharacter`

```prisma
model NovelPromotionCharacter {
  id                      String    @id @default(uuid())
  novelPromotionProjectId String
  name                    String
  aliases                 String?   @db.Text        // JSON: ["别名1", "别名2"]
  introduction            String?   @db.Text        // 角色介绍（身份/关系/称呼映射）

  // ── 视觉相关 ──
  profileData             String?   @db.Text        // JSON: CharacterProfileData
  profileConfirmed        Boolean   @default(false)
  appearances             CharacterAppearance[]      // 视觉外貌描述 → 生图

  // ── 声音相关（当前状态）──
  customVoiceUrl          String?   @db.Text        // 用户手动上传的参考音频 URL
  customVoiceMediaId      String?                    // 关联 MediaObject
  voiceId                 String?                    // 百炼 voiceId（手动填入）
  voiceType               String?                    // "qwen-designed" | "custom"
}
```

**`profileData` 的 TypeScript 结构** (`src/types/character-profile.ts`)：

```typescript
interface CharacterProfileData {
  role_level: "S" | "A" | "B" | "C" | "D";
  archetype: string; // "霸道总裁"
  personality_tags: string[]; // ["高冷", "腹黑"]
  era_period: string; // "现代都市"
  social_class: string; // "上层精英"
  occupation?: string; // "企业家"
  costume_tier: 1 | 2 | 3 | 4 | 5;
  suggested_colors: string[];
  primary_identifier?: string; // "眼角泪痣"
  visual_keywords: string[]; // ["精英气质", "禁欲系"]
  gender: string; // "男"
  age_range: string; // "约二十五岁"
}
```

**问题**：`profileData` 中有丰富的人设信息（gender、age_range、personality_tags、archetype），但**没有任何字段**描述角色的声音特征。`customVoiceUrl` / `voiceId` 与 profileData 之间没有自动化映射——全靠用户手动操作。

```prisma
  // ── 音频相关 ──

  audioDuration       Int?                           // 毫秒

  // ── 镜头匹配（视频专用）──
```

**前端 TypeScript 类型** (`voice-stage-runtime/types.ts`)：

```typescript
interface VoiceLine {
  id: string;
  lineIndex: number;
  speaker: string;
  content: string;
  emotionPrompt: string | null; // 仅用户手动
  emotionStrength: number | null; // 仅标量 0.1-0.5
  audioUrl: string | null;
  matchedPanelId?: string | null;
  matchedStoryboardId?: string | null;
```

**问题**：

1. `emotionStrength` 只有强度没有类型（0.3 可能是愤怒也可能是悲伤）
2. `emotionPrompt` 不由 AI 自动生成，完全依赖用户手动输入
3. 缺少语速/音量/音高等韵律控制维度
4. 缺少台词类型字段（对话 / 旁白 / 内心独白 — 有声书需区分）

## 二、数据存储改造方案

### 2.1 `NovelPromotionCharacter` — 新增声音人设字段

```prisma
model NovelPromotionCharacter {
  // ... 现有字段保持不变 ...

  // ── 🆕 声音人设字段 ──
  voicePrompt           String?   @db.Text  // AI 生成的音色描述文本
                                             // 如："沉稳的青年男性，音色低沉有磁性，
                                             //     说话语速偏慢，带有精英气质"
  voicePreviewUrl       String?   @db.Text  // 音色预览音频 URL（设计时生成）
  voicePreviewMediaId   String?             // 关联 MediaObject
  voiceTargetModel      String?             // 创建音色时使用的 target_model
                                             // 如 "cosyvoice-v3.5-plus"
  voiceDesignConfirmed  Boolean   @default(false)  // 音色是否已确认
}
```

**改造说明**：

| 新字段                 | 类型    | 用途                                    | 来源                                 |
| ---------------------- | ------- | --------------------------------------- | ------------------------------------ |
| `voicePrompt`          | Text    | 存储 AI 根据 profileData 生成的声音描述 | LLM (`agent_character_voice` Prompt) |
| `voicePreviewUrl`      | Text    | 声音设计 API 返回的预览音频             | CosyVoice / QwenVD voice-enrollment  |
| `voicePreviewMediaId`  | String? | 预览音频的 MediaObject 关联             | 上传到存储后获得                     |
| `voiceTargetModel`     | String? | 记录音色绑定的 TTS 模型                 | 创建时指定                           |
| `voiceDesignConfirmed` | Boolean | 用户是否确认了 AI 设计的音色            | 用户操作                             |

**为什么不复用现有 `voiceId` / `voiceType`？**  
可以复用。`voiceId` 继续存储最终确认的音色 ID，`voiceType` 扩展为 `qwen-designed | cosyvoice-designed | custom | uploaded`。新增字段是补充音色**设计过程**的中间数据。

**`profileData` 是否需要改？**  
不需要修改 `CharacterProfileData` 类型本身。声音信息存在独立字段中而非嵌入 profileData，因为：

1. profileData 是 LLM 一次性生成的角色档案，不应频繁修改
2. 音色设计是独立流程，可能多次重新设计
3. 保持与视觉档案的对称性（视觉也是独立的 `CharacterAppearance` 表）

### 2.2 `NovelPromotionVoiceLine` — 扩展情绪与韵律字段

```prisma
model NovelPromotionVoiceLine {
  // ... 现有字段保持不变 ...

  // ── 🆕 情绪类型与韵律控制 ──
  emotionType           String?             // happy|angry|sad|afraid|surprised|calm|...
  emotionInstruction    String?   @db.Text  // 自然语言情绪指令（自动生成）
                                             // 如："愤怒地咆哮，语速加快，音量提高"
  emotionalStrengths    String?   @db.Text  // JSON: IndexTTS2 八维情绪向量
                                             // {"happy":0,"angry":0.8,"sad":0,...}

  // ── 🆕 韵律控制参数 ──
  speakingRate          Float?              // 语速倍率 0.5-2.0（CosyVoice rate）
  pitchRate             Float?              // 音高倍率 0.5-2.0（CosyVoice pitch）
  volumeLevel           Int?                // 音量 0-100（CosyVoice volume）

  // ── 🆕 台词类型（有声书需要区分）──
  lineType              String?   @default("dialogue")
                                             // dialogue | narration | voiceover | inner_thought
}
```

**改造说明**：

| 新字段               | 类型    | 用途                         | 写入时机                             |
| -------------------- | ------- | ---------------------------- | ------------------------------------ |
| `emotionType`        | String? | 情绪分类标签                 | voice_analysis Prompt 自动输出       |
| `emotionInstruction` | Text?   | CosyVoice `instruction` 参数 | voice_analysis Prompt 自动生成       |
| `emotionalStrengths` | Text?   | IndexTTS2 八维向量 JSON      | voice_analysis Prompt 自动生成       |
| `speakingRate`       | Float?  | 语速控制                     | voice_analysis 建议 + 用户手动调整   |
| `pitchRate`          | Float?  | 音高控制                     | voice_analysis 建议 + 用户手动调整   |
| `volumeLevel`        | Int?    | 音量控制                     | voice_analysis 建议 + 用户手动调整   |
| `lineType`           | String? | 台词类型                     | voice_analysis / screenplay 阶段确定 |

**与现有字段的关系**：

```
emotionStrength (现有, Float 0.1-0.5)
  ├── 视频流程: 继续使用，传给 IndexTTS2 的 strength
  └── 有声书流程: 可选使用，或由 emotionalStrengths 取代

emotionPrompt (现有, Text)
  ├── 视频流程: 用户手动填写，传给 IndexTTS2 的 emotion_prompt
  └── 有声书流程: 改为由 AI 自动填写（voice_analysis Prompt 生成）

emotionInstruction (🆕, Text)
  └── CosyVoice 专用: 传给 CosyVoice 的 instruction 参数
      与 emotionPrompt 的区别: emotionPrompt 是英文短语给 IndexTTS2,
      emotionInstruction 是中文自然语言给 CosyVoice
```

## 三、后端逻辑改造方案

### 3.1 角色声音人设生成

**新增文件**：

| 文件                                                       | 用途                                                   |
| ---------------------------------------------------------- | ------------------------------------------------------ |
| `lib/prompts/novel-promotion/agent_character_voice.zh.txt` | profileData → voice_prompt 的 Prompt                   |
| `src/lib/workers/handlers/character-voice-design.ts`       | Worker：调用 LLM 生成 voice_prompt → 调用 API 创建音色 |
| `src/lib/providers/cosyvoice/voice-design.ts`              | CosyVoice v3.5-plus 声音设计 API 封装                  |
| `src/lib/providers/cosyvoice/tts.ts`                       | CosyVoice v3.5-plus 语音合成 API 封装                  |

**`agent_character_voice.zh.txt` Prompt 设计要点**：

```
输入：CharacterProfileData JSON（gender, age_range, personality_tags, archetype, era_period, social_class）
输出：voice_prompt 文本（≤500字符，符合声音设计 API 要求）

映射规则：
- gender → "男性" / "女性"
- age_range → "青年, 约25岁" / "中年, 约45岁"
- personality_tags → 情感风格（["高冷","腹黑"] → "沉稳冷静"）
- archetype → 音色特征（"霸道总裁" → "低沉有磁性，说话有力度"）
- social_class → 说话气质（"上层精英" → "吐字清晰，用词考究"）
- era_period → 风格修正（"古代" → 避免现代口吻描述）

输出示例：
"沉稳的青年男性，约25岁，音色低沉有磁性，富有力度，
 语速偏慢，说话带有精英气质，吐字清晰，适合演绎高冷腹黑角色。"
```

**流程**：

```
handleCharacterVoiceDesign(job)
  1. 读取 character.profileData
  2. LLM(agent_character_voice Prompt) → voice_prompt
  3. 存入 character.voicePrompt
  4. voice-enrollment API(voice_prompt, target_model) → voice_id + preview_audio
  5. 上传 preview_audio → voicePreviewUrl
  6. 存入 character.voiceId, voiceTargetModel
  7. 若自动模式，标记 voiceDesignConfirmed = true
```

```typescript

emotionInstruction: typeof lineData.emotionInstruction === 'string' ? lineData.emotionInstruction : null,
emotionalStrengths: lineData.emotionalStrengths ? JSON.stringify(lineData.emotionalStrengths) : null,
```


### 3.4 有声书特有：旁白台词提取

现有 `voice_analysis.zh.txt` 明确排除旁白（只提取引号内对话）。有声书需要把 screenplay 中的 `action` 和 `voiceover` 也转为台词。

方案：新增 `voice_analysis_audiobook.zh.txt` Prompt 或通过参数控制：

```diff
// voice_analysis.zh.txt 现有规则
- ❌ 严格排除：叙述性文字、动作描写、场景描述

// voice_analysis_audiobook.zh.txt 有声书规则
+ ✅ 对话台词 → lineType: "dialogue"
+ ✅ 旁白/叙述 → lineType: "narration", speaker: "旁白"
+ ✅ 内心独白 → lineType: "inner_thought", speaker: 对应角色
+ ❌ 仅排除：章节标题、纯场景过渡语
```

### 5.2 推荐集成方案

**主方案：CosyVoice v3.5-plus 一站式**

```
角色人设 → voice_prompt → CosyVoice 声音设计 → voice_id
逐句合成 → CosyVoice(voice=voice_id, instruction=情绪, rate/pitch/volume)
```

优势：一个模型同时解决音色创造 + 情绪控制 + 韵律调节。

**补充方案：IndexTTS2 精细情绪**

对于 8 类离散情绪需要精确量化控制的场景（如多角色广播剧），使用 CosyVoice 预览音频作为 IndexTTS2 的参考音频，用 `emotional_strengths` 八维向量做精调。

### 5.3 新增 Provider 文件

```
src/lib/providers/cosyvoice/
├── voice-design.ts       ← 声音设计 API (voice-enrollment + create_voice + query_voice)
├── tts.ts                ← 语音合成 API (WebSocket SpeechSynthesizer)
└── index.ts              ← 统一导出
```

**声音设计 API 封装**（`voice-design.ts`）：

```typescript
interface CosyVoiceDesignInput {
  voicePrompt: string; // ≤500字符
  previewText: string;
  prefix?: string; // 音色名前缀
  language?: "zh" | "en";
  targetModel?: string; // 默认 'cosyvoice-v3.5-plus'
}

interface CosyVoiceDesignResult {
  success: boolean;
  voiceId?: string; // voice_id，用于后续合成
  previewAudioBase64?: string;
  sampleRate?: number;
  requestId?: string;
  error?: string;
}

// API 端点: POST /api/v1/services/audio/tts/customization
// model: "voice-enrollment"
// input.action: "create_voice"
// input.target_model: "cosyvoice-v3.5-plus"
```

**语音合成 API 封装**（`tts.ts`）：

```typescript
interface CosyVoiceTTSInput {
```
